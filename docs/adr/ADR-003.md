# ADR-003: HMAC JWT for Authentication

## Status
Accepted

## Context
The daemon API needs a stateless way to authenticate users and remote workers (fleet nodes). Options included API keys stored in DB, stateful sessions, asymmetric RSA JWTs, and symmetric HMAC JWTs.

## Decision
We chose HMAC (HS256) JWTs for authentication.

## Consequences
- **Pros:**
  - Sufficient security for a single-operator environment or private Tailscale network.
  - Stateless verification; the server doesn't need to perform DB lookups for every request.
  - RBAC is handled efficiently via claims (`role`).
- **Cons:**
  - Symmetric key means all verifiers must also possess the signing key. Not suitable for an environment where verifiers are untrusted, but in Maxwell Daemon, all API serving nodes are trusted coordinators.
