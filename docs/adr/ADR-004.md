# ADR-004: In-Memory Task Queue

## Status
Accepted

## Context
Tasks submitted to Maxwell Daemon need to be queued before execution. Options included Redis, RabbitMQ, and an in-memory queue backed by SQLite.

## Decision
We chose an in-memory `asyncio.PriorityQueue` backed by SQLite for durability.

## Consequences
- **Pros:**
  - Ultra-low latency for the hot path (worker polling).
  - No external infrastructure required (Redis/RabbitMQ).
  - High availability and recovery is achieved by re-loading pending tasks from SQLite on startup.
- **Cons:**
  - Bound by the memory available to the coordinator node.
  - Multi-coordinator active-active setups are complex, but our architecture relies on a single coordinator pattern.
