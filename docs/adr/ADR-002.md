# ADR-002: Subprocess Sandbox

## Status
Accepted

## Context
Code generated by LLMs must be executed in a safe environment to prevent arbitrary code execution on the host machine. We considered Docker, Firecracker VMs, and local subprocesses.

## Decision
We chose a local subprocess execution model with specific shell restrictions for our default sandbox.

## Consequences
- **Pros:**
  - Simplicity. No need for the user to install or manage Docker or virtualization software.
  - Transparent file access for local repository development.
  - Low latency for execution.
- **Cons:**
  - Provides only basic isolation. It is not suitable for running multi-tenant or untrusted external code without a secure perimeter. It is intended for a single-operator environment running against trusted repositories.
