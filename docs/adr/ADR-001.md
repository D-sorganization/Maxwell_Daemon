# ADR-001: SQLite for Persistence

## Status
Accepted

## Context
Maxwell Daemon needs durable storage for tasks, costs, workflows, and artifacts. Options considered included PostgreSQL, Redis, and SQLite.

## Decision
We chose SQLite for all persistence needs (tasks, artifacts, actions, costs).

## Consequences
- **Pros:**
  - Zero operational overhead. No need to install, configure, or run a separate database server.
  - Portable and easy to backup (just copy the file).
  - Perfect for a home-user daemon intended to run on local hardware or as a single container.
- **Cons:**
  - Not suited for massive concurrent writes (handled by connection pooling and WAL mode).
  - Multi-node scaling requires a dedicated coordinator node since SQLite is local to the filesystem.
