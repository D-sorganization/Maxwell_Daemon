# ADR-005: Design-by-Contract

## Status
Accepted

## Context
Codebase safety and reliability can be handled via defensive programming, pervasive try-except blocks, or strict contracts.

## Decision
We enforce a design-by-contract approach across the Maxwell Daemon architecture. Invariants and preconditions are checked immediately using assertions or explicit `ValueError`/`RuntimeError` exceptions.

## Consequences
- **Pros:**
  - Fail-fast behavior prevents data corruption and makes bugs easier to trace.
  - Reduces the need for deeply nested `if-else` error handling logic in the core execution loop.
- **Cons:**
  - An unexpected failure crashes the task (or daemon component) completely, requiring the orchestrator to catch and manage the crash, which is why we wrap top-level executions in robust exception handlers.
